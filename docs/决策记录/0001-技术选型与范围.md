# 决策记录 0001：技术选型与第一版范围

## 状态

已确认，2026-07-15生效。

## 背景

文书扫描任务通常为数百页，需要在 Windows客户机通过 KODAK i2000系列 TWAIN Data Source 控制扫描仪、保存原图和缩略图，并允许调用方在后续流程中读取文件。当前真机为 KODAK i2400。

## 决策

- 创建独立项目 `D:\archive-scan-service`，不把 Python代码放入 Vue工程。
- 使用 Windows 64位和 Python 64位。
- 仅支持64位 TWAIN，使用64位 `TWAINDSM.DLL`。
- 使用 `pytwain 2.3.0`，接受 GPLv2交付要求。
- 使用登录用户级托盘程序，不注册 Windows Service。
- 使用 FastAPI主进程和长期 TWAIN工作子进程。
- 仅使用 `TWSX_FILE`，固定单页 JPEG/JFIF。
- 禁止厂商界面，固定业务配置字段通过 Capability查询、参数校验和设置。
- 查询全部标准和私有 Capability。
- Python不修改原图，只允许生成缩略图。
- 使用 SQLite和 `%LOCALAPPDATA%\ArchiveScanService`。
- 固定监听 `127.0.0.1:17653`。
- 第一版不做身份授权、配对码和访问令牌。

## 取舍

选择双进程是为了隔离 TWAIN驱动卡死和异常退出；选择文件传输是为了避免数百页原图长期进入 Python内存；选择独立项目是为了分离前端部署、客户机安装、许可证和版本生命周期。

## 后果

- 不支持文件传输或 JPEG输出的扫描设备不兼容第一版。
- `pytwain`缺失的 `MSG_QUERYSUPPORT`和通用设置能力需要项目内扩展。
- 客户试用也必须履行 GPLv2许可证和对应源代码义务。
- 真实设备兼容结论必须通过实际使用 `kds_i2000.inf` 的目标真机测试获得；具体扫描能力以 `KODAK Scanner: i2000` Data Source 的实时 Capability 为准。

## 2026-07-21 设计修订

- 前端不根据 `CAP_SUPPORTEDCAPS` 返回的 156 项 Capability 动态生成配置表，改为使用固定业务配置字段，并通过标准名称和 Capability 编号映射到 TWAIN 参数。
- Capability 查询仍返回全部标准和私有参数，用于判断字段是否存在、候选值、范围、当前值、默认值和查询错误。
- `MSG_QUERYSUPPORT` 返回的 `TWQC_SET`、`TWQC_GETCURRENT` 和 `TWQC_RESET` 操作位必须保留，但不作为固定配置字段的唯一显示或设置门槛；真实设置以 `MSG_SET` 返回状态和实际扫描结果为准。
- `MSG_GETCURRENT` 可用时回读实际值；不可用时记录无法回读，不把缺少回读单独判定为设置失败。
