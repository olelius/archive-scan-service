# 决策记录 0001：技术选型与第一版范围

## 状态

已确认，2026-07-15生效。

## 背景

文书扫描任务通常为数百页，需要在 Windows客户机连接 KODAK i2600，通过本机程序控制扫描仪、保存原图和缩略图，并允许调用方在后续流程中读取文件。

## 决策

- 创建独立项目 `D:\archive-scan-service`，不把 Python代码放入 Vue工程。
- 使用 Windows 64位和 Python 64位。
- 仅支持64位 TWAIN，使用64位 `TWAINDSM.DLL`。
- 使用 `pytwain 2.3.0`，接受 GPLv2交付要求。
- 使用登录用户级托盘程序，不注册 Windows Service。
- 使用 FastAPI主进程和长期 TWAIN工作子进程。
- 仅使用 `TWSX_FILE`，固定单页 JPEG/JFIF。
- 禁止厂商界面，所有参数通过 Capability查询和设置。
- 查询全部标准和私有 Capability。
- Python不修改原图，只允许生成缩略图。
- 使用 SQLite和 `%LOCALAPPDATA%\ArchiveScanService`。
- 固定监听 `127.0.0.1:17653`。
- 第一版不做身份授权、配对码和访问令牌。

## 取舍

选择双进程是为了隔离 TWAIN驱动卡死和异常退出；选择文件传输是为了避免数百页原图长期进入 Python内存；选择独立项目是为了分离前端部署、客户机安装、许可证和版本生命周期。

## 后果

- 不支持文件传输或 JPEG输出的扫描设备不兼容第一版。
- `pytwain`缺失的 `MSG_QUERYSUPPORT`和通用设置能力需要项目内扩展。
- 客户试用也必须履行 GPLv2许可证和对应源代码义务。
- 真实设备兼容结论必须通过 KODAK i2600实机测试获得。
